Markdown¶

La langage Markdown est un système de formatage de texte. Il est similaire au format html utilisé pour éditer les pages hébergées sur un serveur web et le format reStructuredText que l’on retrouve dans les wikis.

Le format html utilise un système de balises qui rend le texte peu lisible. Le format rst est plus lisible avec son système de balisage plus léger. Il est utilisé notamment pour la documentation du langage Python mais son utilisation est moins répandu que le markdown.

La documentation des fonctions Julia est en markdown. Il n’y a pas de système de documentation intégré pour la langage R mais roxygen2 est le package utilisé. La syntaxe a des similarités avec LaTeX

Les fichiers markdown sont identifiés par la terminaison .md.

Dans les exemples qui vont suivre, je présente les bases du langage markdown. Il faut savoir qu’il existe un grand nombre de dialectes. Github propose des fonctionalités enrichies. Pour voir toutes les possibilités, je conseille d’aller regarder la démonstration de l’éditeur en ligne codimd.

Code R¶

y <-  c(1,2,3,4,5)
x0 <- c(1,1,1,1,1)  
x1 <- c(1,2,3,4,5)
x2 <- c(1,4,5,7,9)
Y <- as.matrix(y)
X <- as.matrix(cbind(x0,x1,x2))
beta = solve(crossprod(X), crossprod(X,y))

Code Python¶

import numpy as np
y =  np.array((1,2,3,4,5))
x0 = np.array((1,1,1,1,1)) 
x1 = np.array((1,2,3,4,5))
x2 = np.array((1,4,5,7,9))
X = np.c_[x0,x1,x2]
Y = y[:, np.newaxis]
beta = np.linalg.solve(X.T @ X, X.T @ Y)

Code Julia¶

using LinearAlgebra
y =  [1,2,3,4,5]
x0 = [1,1,1,1,1]
x1 = [1,2,3,4,5]
x2 = [1,4,5,7,9]
X = hcat(x0,x1,x2)
Y = y
beta = X'X \ X'y
